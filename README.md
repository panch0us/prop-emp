# IC Web

Внутренний Django-проект для учета сотрудников, имущества, информационных систем и связанных служебных данных.

Проект помогает вести рабочие справочники в одном месте: карточки сотрудников, закрепленное имущество, сетевую технику, информационные системы, семейные сведения сотрудников и выгрузки в Excel.

## Возможности

- Учет сотрудников, должностей, подразделений, званий и контактных данных.
- Учет имущества с владельцами, кабинетами, серийными номерами, гарантией и состоянием.
- Связь имущества между собой: например HDD -> сервер -> стойка.
- История действий с имуществом: установка, ремонт, перемещение, списание и другие события.
- Учет информационных систем и оборудования, которое входит в их состав.
- Учет техники, подключенной к сети, включая физические и виртуальные IP-адреса.
- Inline-редактирование связанных данных в карточке сотрудника: дети, супруги, личный автотранспорт и личное оружие.
- Главная страница с ближайшими днями рождения сотрудников.
- Светлая и темная тема сайта с сохранением выбора пользователя.
- Excel-выгрузки для сотрудников, имущества, АРМ, информационных систем и связанных списков.

## Основные разделы

- `Сотрудники` — карточки сотрудников и связанные сведения.
- `Имущество` — учет оборудования, владельцев, статусов и действий.
- `Информационные системы` — карточки ИС и привязанное к ним оборудование.
- `Техника, подключенная к сети` — сетевые параметры оборудования.
- `Списки для скачивания` — XLS/XLSX-выгрузки для работы и отчетности.

## Стек

- Python 3.10
- Django 4.1
- PostgreSQL
- Gunicorn
- XlsxWriter
- xlwt

## Локальный запуск

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Если файл `requirements.txt` отсутствует, зависимости нужно установить согласно версии проекта и окружения сервера.

## Проверка перед запуском

```bash
python manage.py check
python manage.py migrate
```

Для production-окружения после обновления кода обычно требуется перезапуск WSGI/Gunicorn-сервиса.

## Примечания

Проект рассчитан на внутреннее использование в закрытой сети. Перед публикацией репозитория стоит проверить настройки, секреты, дампы, медиафайлы и служебные данные, чтобы случайно не разместить приватную информацию.
